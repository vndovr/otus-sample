Otus example application

Backend for frontends. Apigateway.

Add authentication and registration for users

Implement a use case "Change and view data in user's profile".

After user is registered he is able to get his profile data from a specific url. It is accessible only for this user and not available for other users

Outcome

  1. Representation of architecture solution and interaction between services
  2. Installation guide (from helm). Specify a namespace
  3. Postman tests that covers following use-cases
    • registration of user 1
    • verification that profile can not be accessed without login
    • login of user 1
    • change the profile for user 1
    • verification that profile has changed
    • logout of user 1
    • registration of user 2
    • login of user 2
    • verification that user 2 has no access to edit profile of user 1

What is important

  • usage for {{baseUrl}} as parameter
  • arch.homework as an initial value
  • random data in tests scenarios
  • output of request/response data
